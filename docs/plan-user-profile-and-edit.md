# Plan: User Profile & Profile Edit

**PRD:** docs/prd-user-profile-and-edit.md
**Дата:** 2026-05-19

## Фазы реализации

---

### Фаза 1: Backend — Core Profile API (Tracer Bullet)

**Цель:** Минимальный рабочий бэкенд: читать и обновлять имя пользователя через API.
**Затрагивает:** database, backend

**Задачи:**

- [ ] Добавить поля `name String?` и `avatarUrl String?` в Prisma-модель `User`, создать и применить миграцию
- [ ] Добавить команды/запросы CQRS: `GetMeQuery` / `GetMeHandler` и `UpdateProfileCommand` / `UpdateProfileHandler`
- [ ] Добавить `UsersController` с эндпоинтами `GET /users/me` и `PATCH /users/me` (JWT-guard, возвращает `{ id, email, name, avatarUrl }`)
- [ ] Добавить `UpdateProfileDto` с валидацией (`name` — строка, опционально)
- [ ] Написать e2e-тесты: `GET /users/me` (200 + структура ответа), `PATCH /users/me` (200 + обновление имени, 400 при невалидном теле)

**Когда готова:** `GET /users/me` и `PATCH /users/me` возвращают корректные данные, e2e-тесты проходят.

---

### Фаза 2: Backend — Avatar Upload & Password Change

**Цель:** Полный бэкенд API профиля: загрузка аватара и смена пароля.
**Затрагивает:** backend

**Задачи:**

- [ ] Добавить эндпоинт `POST /users/me/avatar` (multer, ограничения: 5 МБ, image/jpeg|png|webp) — сохраняет файл в `uploads/avatars/`, обновляет `avatarUrl` в БД, возвращает `{ avatarUrl }`
- [ ] Добавить команду `ChangePasswordCommand` / `ChangePasswordHandler` с bcrypt-проверкой старого пароля
- [ ] Добавить эндпоинт `POST /users/me/change-password` с `ChangePasswordDto` (`oldPassword`, `newPassword`)
- [ ] Отдавать статические файлы из `uploads/` через NestJS `ServeStaticModule` или `useStaticAssets`
- [ ] Написать e2e-тесты: загрузка аватара (200 + avatarUrl), смена пароля (200 при верном / 400 при неверном старом пароле)

**Когда готова:** Аватар загружается и доступен по URL, пароль меняется с проверкой старого — e2e-тесты проходят.

---

### Фаза 3: Frontend — Profile Pages

**Цель:** Страницы `/profile` и `/profile/edit` с полным функционалом редактирования.
**Затрагивает:** frontend

**Задачи:**

- [ ] Создать страницу `/profile` (read-only): аватар, имя или email, email, дата регистрации, кнопка «Редактировать профиль» → `/profile/edit`
- [ ] Создать страницу `/profile/edit` с секцией **Имя** (текстовое поле, кнопка «Сохранить») и секцией **Смена пароля** (старый пароль, новый пароль × 2, кнопка «Сменить пароль»)
- [ ] Добавить секцию **Аватар** на `/profile/edit`: drag-and-drop загрузка + превью текущего аватара, кнопка «Загрузить»
- [ ] Добавить вспомогательную функцию `getNameFromToken` в `lib/auth.ts` и хук/запрос `fetchProfile` к `GET /users/me`
- [ ] Проверить все пути вручную: успешное сохранение, ошибки валидации, неверный пароль, unsupported file type

**Когда готова:** Пользователь может посмотреть профиль, изменить имя, загрузить аватар и сменить пароль через UI.

---

### Фаза 4: Frontend — Header Integration

**Цель:** Аватар и имя пользователя отображаются в шапке главной страницы.
**Затрагивает:** frontend

**Задачи:**

- [ ] Добавить запрос `GET /users/me` в `home-page.tsx` (параллельно с загрузкой встреч)
- [ ] Заменить отображение email в шапке на аватар (если есть) + имя (если есть, иначе email)
- [ ] Сделать блок с аватаром/именем кликабельным — ссылка на `/profile`
- [ ] Проверить: после обновления профиля на `/profile/edit` и возврата на главную шапка показывает актуальные данные

**Когда готова:** Шапка главной страницы отображает актуальный аватар и имя пользователя, ссылается на профиль.
