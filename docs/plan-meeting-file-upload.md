# Plan: Meeting File Upload

**PRD:** docs/prd-meeting-file-upload.md
**Дата:** 2026-05-15

## Фазы реализации

### Фаза 1: Database + Core Upload API (Tracer Bullet)

**Цель:** Минимальный рабочий путь — файл можно загрузить и получить список файлов встречи через API
**Затрагивает:** database, backend
**Задачи:**

- [ ] Написать E2E тесты: загрузка файла → файл появляется в списке; загрузка недопустимого типа/размера → 400
- [ ] Добавить модель `MeetingFile` в Prisma-схему и применить миграцию
- [ ] Настроить `UPLOAD_DIR` env-переменную и автосоздание директории при старте
- [ ] Реализовать `POST /meetings/:id/files` — приём multipart, сохранение на диск, запись метаданных в БД
- [ ] Реализовать `GET /meetings/:id/files` — возврат списка файлов встречи с метаданными; подключить валидацию типа и размера

**Когда готова:** E2E тесты фазы зелёные; `POST` принимает валидный файл, файл появляется на диске, `GET` возвращает его метаданные

---

### Фаза 2: Download & Delete API

**Цель:** Полный CRUD через API — файл можно скачать и удалить
**Затрагивает:** backend
**Задачи:**

- [ ] Написать E2E тесты: скачивание файла → корректный `Content-Disposition`; удаление владельцем → 200; удаление чужого файла → 403
- [ ] Реализовать `GET /meetings/:id/files/:fileId/download` — стриминг файла через `StreamableFile` с корректным `Content-Disposition`
- [ ] Реализовать `DELETE /meetings/:id/files/:fileId` — удаление файла с диска и из БД
- [ ] Добавить ownership-check на DELETE: только владелец встречи может удалять файлы
- [ ] Покрыть все эндпоинты `JwtGuard`

**Когда готова:** E2E тесты фазы зелёные; все четыре эндпоинта работают корректно; попытка удаления чужого файла возвращает 403

---

### Фаза 3: Frontend — Files Panel (список и скачивание)

**Цель:** Пользователь видит список файлов встречи и может скачать любой файл
**Затрагивает:** frontend
**Задачи:**

- [ ] Написать E2E тесты: страница встречи содержит секцию «Файлы»; клик по файлу запускает скачивание; владелец видит кнопку удаления, не-владелец — нет
- [ ] Добавить вкладку/секцию «Файлы» на страницу встречи
- [ ] Реализовать отображение списка файлов: имя, размер, дата загрузки, иконка по MIME-типу
- [ ] Подключить скачивание файла по клику через эндпоинт `/download`
- [ ] Показывать кнопку удаления только владельцу встречи; реализовать удаление без перезагрузки страницы

**Когда готова:** E2E тесты фазы зелёные; страница встречи отображает список файлов, клик скачивает файл, владелец может удалять файлы

---

### Фаза 4: Frontend — Upload UX (drag-and-drop + прогресс)

**Цель:** Удобный интерфейс загрузки файлов с визуальной обратной связью
**Затрагивает:** frontend
**Задачи:**

- [ ] Написать E2E тесты: drag-and-drop загрузка → файл появляется в списке; загрузка недопустимого файла → сообщение об ошибке отображается
- [ ] Добавить drag-and-drop зону загрузки файла
- [ ] Реализовать прогресс-бар во время загрузки
- [ ] Показывать понятное сообщение об ошибке при недопустимом типе или размере файла
- [ ] Обновлять список файлов после успешной загрузки без перезагрузки страницы

**Когда готова:** E2E тесты фазы зелёные; пользователь может перетащить файл в зону, видит прогресс, файл появляется в списке после загрузки
